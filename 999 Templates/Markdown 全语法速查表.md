## 一、 标题（分层逻辑）

在文字前加 `#`，**注意：# 后必须有空格。**

`# 一级标题（通常用于笔记大标题）`
# 一级
`## 二级标题（用于主要章节）`
## 二级
`### 三级标题（用于子要点）`
### 三级
`#### 四级标题（以此类推）`
#### 四级

## 二、 文字样式（视觉重点）

- **加粗**：`**这里是粗体**`
    **加粗**
- _斜体_：`*这里是斜体*`
    *斜体*
- **_粗斜体_**：`***这里是粗斜体***`
    ***粗斜体***
- ~~删除线~~：`~~删掉的内容~~`
    ~~删除线~~
- ==高亮==：`==最核心的重点==`（Obsidian 特有）
    ==高亮==
- 脚注：`内容[^1]`（会自动在文末生成引文）
    [^1]

# 补充
### 搜索：在知识海洋中精准定位

当笔记数量达到几百上千条时，搜索功能就成了救命稻草。Obsidian 的搜索支持全文检索，你可以用关键词、标签、文件名等多种方式查找。

更强大的是搜索运算符。比如：

- `tag:# 时间管理` 只搜索带这个标签的笔记
- `file：卡片盒` 只搜索文件名包含「卡片盒」的笔记
- `“精确短语”` 搜索完全匹配的内容

这些功能让你在不记得笔记位置时，依然能快速找到需要的信息。

### 最重要的规则

卡片盒笔记法只有一条铁律：**每条永久笔记至少要链接到另一条笔记**。

这个规则看似简单，却蕴含深意。它强迫你思考：这个新知识和我已知的哪些内容有关？这种主动建立连接的过程，就是深度学习的本质。

当你写一条关于「复利效应」的笔记时，你可能会链接到「指数增长」「习惯养成」「投资策略」等笔记。这些链接不是装饰，而是你思维路径的记录。几个月后，当你打开关系图谱，会惊讶地发现某些概念成了枢纽节点，这些就是你知识体系的核心支柱。



## 代办

### 基础必备插件

- [ ] **Calendar**：在侧边栏显示日历，点击日期可快速创建或打开当天的日记。如果你有写日记的习惯，这个插件能让你一键回顾过去任何一天的记录。

- [ ] **Periodic Notes**：扩展日记功能，支持创建周记、月记、年记。配合模板使用，可以建立定期复盘机制——每周回顾本周学到了什么，每月总结进展，每年规划目标。

- [ ] **Recent Files**：在侧边栏显示最近打开的文件列表。当你频繁在几条笔记间切换时，这个插件能大幅提升效率。

- [ ] **Tag Wrangler**：增强标签管理功能，支持批量重命名标签、合并标签、查看标签层级。当你的标签系统变得复杂时，这个工具能帮你保持整洁。

### 进阶生产力插件

- [ ] **Dataview**：这是 Obsidian 的「数据库」插件。你可以用类似 SQL 的语法查询笔记，比如「列出所有标记为 # 待办 且创建时间在本周的笔记」。适合喜欢结构化管理的用户。

- [ ] **[Tasks](https://zhida.zhihu.com/search?content_id=270691231&content_type=Article&match_order=1&q=Tasks&zhida_source=entity)**：增强任务管理功能，支持设置截止日期、优先级、重复任务。如果你想在 Obsidian 中管理待办事项，这个插件是最佳选择。
